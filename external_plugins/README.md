# 🧩 Dify Chat Tester 外部插件目录

欢迎使用插件系统！此目录用于存放和管理自定义扩展插件，让您的 Dify Chat Tester 功能更加强大。

---

## 📂 支持的格式

我们支持以下两种方式安装插件，程序启动时会自动识别并加载：

| 图标 | 格式       | 适用场景  | 说明                                                       |
| :--: | :--------- | :-------- | :--------------------------------------------------------- |
|  📁  | **文件夹** | 开发/调试 | 直接将插件源码目录放入此处，改动即时生效。                 |
|  📦  | **ZIP 包** | 生产/分发 | 直接放入 `.zip` 压缩包，程序启动时会自动解压并从内存加载。 |

## 🚀 快速开始

### 方式一：安装源码插件 (文件夹)

1.  将插件文件夹直接拷贝到本目录：
    ```text
    external_plugins/
    └── my_awesome_plugin/
        ├── __init__.py
        ├── provider.py
        └── ...
    ```
2.  重启 `dify_chat_tester`，新插件即会自动出现在供应商列表中。

### 方式二：安装离线包 (ZIP)

1.  获取插件提供的 `.zip` 安装包（例如 `my_plugin_v0.1.0.zip`）。
2.  将其直接放入本目录：
    ```text
    external_plugins/
    └── my_plugin_v0.1.0.zip
    ```
3.  重启程序，系统会自动处理解压和加载。

---

## 🛠️ 依赖管理

如果您的插件需要第三方 Python 库（例如 `requests`, `bs4` 等）：

1.  在插件根目录下创建 `requirements.txt` 文件。
2.  **自动检测**：程序加载插件时会自动检查这些依赖。
3.  **智能提示**：
    - **源码运行模式**：如果有安装 `uv`，程序会询问是否**自动安装**缺失依赖。
    - **打包运行模式**：程序会友好提示缺失的库及手动安装命令。

## ⚙️ 路径配置

如果您希望将插件目录放置在其他位置（例如为了方便备份），可以在主程序的 `.env.config` 文件中指定路径：

```ini
# 指定外部插件的绝对路径或相对路径
EXTERNAL_PLUGINS_PATH=/path/to/my/custom/plugins
```
