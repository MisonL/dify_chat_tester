## 3. 🐍 安装 Python

### 🎯 什么是 Python？

Python 是一种流行的编程语言，本程序需要 Python 环境才能运行。

### 📥 安装步骤：

1. **🌐 访问官网**

   - 打开浏览器，访问：https://www.python.org/downloads/
   - 网站会自动检测您的系统并推荐合适的版本

2. **⬇️ 下载安装包**

   - 点击黄色的 "Download Python x.x.x" 按钮
   - 会下载 `python-x.x.x-amd64.exe` 文件

3. **⚙️ 安装 Python**

   - 直接点击 "Install Now" 进行默认安装
   - **ℹ️ 注意**：大多数用户会直接点击下一步，不会勾选 "Add Python to PATH"
   - 等待安装完成

4. **✅ 验证安装**
   - 按 `Win + R` 键，输入 `cmd`，回车打开命令提示符
   - 输入以下命令验证 py 启动器：
     ```bash
     py --version
     ```
   - 如果显示 Python 版本号（如 `Python 3.11.5`），说明安装成功

### 📋 安装后的两种使用方案

Python 安装完成后，有两种使用方案：

#### 方案一：使用 py 启动器（推荐，无需任何配置）

Python 安装包会自动安装 Python 启动器 `py.exe`，它可以自动找到系统中安装的 Python，无需环境变量：

1. **验证 py 启动器可用**

   ```bash
   py --list
   ```

   会列出系统中安装的所有 Python 版本，例如：

   ```
   Installed Pythons found by py Launcher for Windows
    -3.11-64 *
    -3.9-64
   ```

2. **使用 py 启动器运行 Python**

   ```bash
   # 使用默认 Python 版本
   py --version

   # 使用指定版本
   py -3.11 --version

   # 运行 Python 脚本
   py main.py
   ```

3. **创建便捷批处理文件（可选）**

   - 在项目根目录创建 `py_run.bat` 文件：

   ```batch
   @echo off
   py %*
   ```

   - 之后可以使用 `py_run.bat` 代替 `py` 命令

4. **py 启动器的优势**
   - 无需配置环境变量
   - 自动选择合适的 Python 版本
   - 可以指定版本号（如 `py -3.11`）
   - 安装 Python 时自动安装，无需额外配置

#### 方案二：添加 Python 到环境变量（备选方案）

如果您希望直接使用 `python` 命令，可以选择添加环境变量。有两种方法：

##### 方法一：安装 Python 时勾选选项（最简单）

如果您还未安装 Python，可以在安装时直接添加环境变量：

1. 运行 Python 安装程序
2. **重要：勾选 "Add Python to PATH" 选项**
3. 点击 "Install Now" 完成安装
4. 安装后可以直接使用 `python` 命令

##### 方法二：手动添加环境变量（已安装 Python）

如果已经安装了 Python 但未添加环境变量：

##### 为什么要添加环境变量？

- 可以直接使用 `python` 命令
- 符合大多数网络教程的命令格式
- 适合经常使用 Python 的用户

##### 如何手动添加环境变量？

1. **找到 Python 安装路径**

   - 通常在：`C:\Users\您的用户名\AppData\Local\Programs\Python\Pythonxx`
   - 可以通过以下方式找到：
     - 在开始菜单搜索 "Python"
     - 右键点击 → 打开文件位置
     - 右键点击 Python 快捷方式 → 属性
     - 复制"目标"中的路径

2. **打开环境变量设置**

   - 按 `Win + R`，输入 `sysdm.cpl`，回车
   - 点击"高级"标签
   - 点击"环境变量"按钮

3. **编辑 Path 变量**

   - 在"系统变量"部分找到 `Path`，点击"编辑"
   - 点击"新建"，添加以下两条路径：
     - `C:\Users\您的用户名\AppData\Local\Programs\Python\Pythonxx`
     - `C:\Users\您的用户名\AppData\Local\Programs\Python\Pythonxx\Scripts`
   - 点击"确定"保存所有设置

4. **重新打开命令提示符验证**
   ```bash
   python --version
   ```

##### 方案三：使用完整路径（临时方案）

如果不想修改环境变量，也可以直接使用 Python 的完整路径：

1. **找到 Python.exe**

   - 通常在：`C:\Users\您的用户名\AppData\Local\Programs\Python\Pythonxx\python.exe`

2. **使用完整路径运行命令**
   ```bash
   "C:\Users\您的用户名\AppData\Local\Programs\Python\Python311\python.exe" --version
   ```

## 4. 📦 安装 uv 包管理器

### 🎯 什么是 uv？

uv 是一个快速的 Python 包管理器，类似于 npm 或 yarn。

### 📥 安装步骤：

### 📦 安装 uv 的两种方式

#### 方式一：在虚拟环境中安装（推荐）

这种方式将 uv 安装在项目虚拟环境中，每个项目独立，互不影响。

1. **创建虚拟环境**

   ```bash
   # 使用 py 启动器（推荐）
   py -m venv .venv

   # 或如果已配置环境变量
   python -m venv .venv
   ```

2. **激活虚拟环境**

   ```bash
   # cmd 中
   .venv\Scripts\activate.bat

   # PowerShell 中
   .venv\Scripts\Activate.ps1
   ```

3. **在虚拟环境中安装 uv**

   ```bash
   # 升级 pip
   py -m pip install --upgrade pip

   # 安装 uv
   py -m pip install uv
   ```

4. **验证安装**
   ```bash
   uv --version
   ```

**特点**：

- 只在当前虚拟环境中可用
- 不需要管理员权限
- 项目隔离，更安全
- 符合 Python 最佳实践
- 每个项目可以有不同版本的 uv

#### 方式二：全局安装（适合系统管理员）

这种方式将 uv 安装到系统中，所有项目都可以使用。

⚠️ **重要：PowerShell 安全策略设置**

在运行 PowerShell 脚本之前，可能需要调整执行策略：

1. **查看当前执行策略**

   ```bash
   Get-ExecutionPolicy
   ```

2. **如果显示 `Restricted`，需要临时允许脚本执行**

   ```bash
   # 临时允许当前会话执行脚本（推荐）
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope Process

   # 或永久修改（需要管理员权限）
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```

3. **安装完成后可以恢复默认策略**

   ```bash
   Set-ExecutionPolicy -ExecutionPolicy Restricted -Scope Process
   ```

4. **在 VSCode 终端中运行**

   ```bash
   # Windows PowerShell
   powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

   # 或使用 cmd（无需调整策略）
   cmd /c "curl -LsSf https://astral.sh/uv/install.sh | sh"
   ```

5. **验证安装**

   ```bash
   uv --version
   ```

6. **重启终端**
   - 关闭当前终端，重新打开

**执行策略说明**：

- `Restricted`：默认设置，不允许运行任何脚本
- `RemoteSigned`：允许本地脚本运行，下载的脚本需要数字签名
- `Process`：仅影响当前 PowerShell 会话，关闭后恢复
- `CurrentUser`：仅影响当前用户，不需要管理员权限

**特点**：

- 全局安装，所有项目都可以使用
- 可能需要管理员权限
- 系统只有一个 uv 版本
- 适合经常使用 Python 的开发者

### 💡 两种安装方式的区别

| 特点         | 在线脚本安装      | pip 虚拟环境安装 |
| ------------ | ----------------- | ---------------- |
| **安装位置** | 用户目录/系统目录 | 虚拟环境内部     |
| **可用范围** | 全局所有项目      | 仅当前虚拟环境   |
| **权限要求** | 可能需要管理员    | 不需要管理员     |
| **版本管理** | 系统单一版本      | 每个项目可不同   |
| **隔离性**   | 无隔离            | 完全隔离         |
| **推荐场景** | 系统管理员        | 项目开发（推荐） |

**建议**：对于项目开发，推荐使用 pip 在虚拟环境中安装，更安全、更灵活。

### 💡 使用提示

- 如果 `uv` 命令不可用，可以始终使用 `py -m uv` 代替
- VSCode 会自动检测 Python 环境，即使没有 PATH 也能正常工作
- 推荐使用方案一（py 启动器），无需任何配置

## 8. ⚙️ 安装项目依赖

### 📦 配置 pip 镜像源（推荐，提升下载速度）

在安装依赖之前，建议先配置 pip 使用国内镜像源，可以大幅提升下载速度：

1. **激活虚拟环境**

   ```bash
   # cmd 中
   .venv\Scripts\activate.bat

   # PowerShell 中
   .venv\Scripts\Activate.ps1
   ```

   激活后，命令行前面会出现 `(.venv)` 标识

2. **设置阿里云镜像源**

   ```bash
   # 设置全局镜像源
   pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/

   # 设置信任主机
   pip config set global.trusted-host mirrors.aliyun.com
   ```

3. **验证配置**

   ```bash
   pip config list
   ```

   应该看到：

   ```
   global.index-url='https://mirrors.aliyun.com/pypi/simple/'
   global.trusted-host='mirrors.aliyun.com'
   ```

4. **其他可选镜像源**
   - 清华大学：`https://pypi.tuna.tsinghua.edu.cn/simple`
   - 豆瓣：`https://pypi.douban.com/simple`
   - 华为云：`https://mirrors.huaweicloud.com/repository/pypi/simple`

⚠️ **注意**：pip 配置命令需要在虚拟环境中运行，这样配置才会应用到当前项目的 Python 环境。

### 📦 安装所有依赖

根据 uv 的安装方式，有两种不同的依赖安装方法：

#### 方式一：如果使用在线脚本全局安装了 uv

这种方式最简单，uv 会自动创建和管理虚拟环境：

1. **直接安装依赖**

   ```bash
   uv sync
   ```

2. **等待安装完成**

   - uv 会自动创建 `.venv` 虚拟环境
   - 自动安装所有依赖到虚拟环境
   - 看到 "Success" 提示表示安装完成

3. **后续使用**
   - 运行程序时：`uv run python main.py`
   - 或激活环境后：`.venv\Scripts\activate.bat` 然后 `python main.py`

#### 方式二：如果在虚拟环境中安装了 uv

需要手动管理虚拟环境：

1. **创建虚拟环境**

   ```bash
   # 使用 py 启动器（推荐）
   py -m venv .venv

   # 或如果已配置环境变量
   python -m venv .venv
   ```

2. **激活虚拟环境**

   ```bash
   # cmd 中
   .venv\Scripts\activate.bat

   # PowerShell 中
   .venv\Scripts\Activate.ps1
   ```

   激活后，命令行前面会出现 `(.venv)` 标识

3. **安装 uv（如果还没有安装）**

   ```bash
   # 在虚拟环境中安装 uv
   py -m pip install uv
   ```

4. **安装项目依赖**

   ```bash
   uv sync
   ```

5. **等待安装完成**
   - uv 会安装所有依赖到当前虚拟环境
   - 看到 "Success" 提示表示安装完成

### 💡 一键完成脚本（可选）

您可以创建一个批处理文件 `setup.bat` 来一键完成所有设置：

```batch
@echo off
echo 创建虚拟环境...
py -m venv .venv
echo 激活虚拟环境...
call .venv\Scripts\activate.bat
echo 安装 uv...
py -m pip install uv
echo 安装项目依赖...
uv sync
echo 完成！
pause
```

### 🚨 常见问题

- **提示"py 不是内部或外部命令"**：使用 Python 完整路径
- **提示"uv 不是内部或外部命令"**：先运行 `py -m pip install uv`
- **虚拟环境激活失败**：确保使用正确的激活脚本（.bat 或 .ps1）
  ```bash
  uv.bat sync
  ```

#### 情况四：使用完整路径或 python -m

1. **使用完整路径的 uv**

   ```bash
   "C:\Users\您的用户名\AppData\Python\Python311\Scripts\uv.exe" sync
   ```

2. **或使用 python -m uv**
   ```bash
   "C:\Users\您的用户名\AppData\Local\Programs\Python\Python311\python.exe" -m uv sync
   ```

#### 情况三：使用传统 pip 方式（备用方案）

如果 uv 安装有问题，可以使用传统的 pip 方式：

1. **创建虚拟环境**

   ```bash
   # 使用完整路径
   "C:\Users\您的用户名\AppData\Local\Programs\Python\Python311\python.exe" -m venv .venv

   # 或如果 python 命令可用
   python -m venv .venv
   ```

2. **激活虚拟环境**

   ```bash
   # 在 cmd 中
   .venv\Scripts\activate

   # 在 PowerShell 中
   .\.venv\Scripts\Activate.ps1
   ```

3. **升级 pip**

   ```bash
   python -m pip install --upgrade pip
   ```

4. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```
   （如果没有 requirements.txt，可以使用 pyproject.toml）
   ```bash
   pip install -e .
   ```

### ✅ 验证安装

无论使用哪种方式，都可以通过以下命令验证：

```bash
# 检查依赖是否安装成功
python -c "import requests, openpyxl, rich; print('所有依赖安装成功！')"
```

### 💡 使用提示

- 推荐使用 uv 方式，速度更快且更可靠
- 如果遇到权限问题，可以尝试使用管理员权限运行终端
- VSCode 会自动检测虚拟环境，通常不需要手动激活

## 9. 🎮 程序使用方法

### 🧩 使用打包好的程序（推荐，免安装环境）

如果您是**普通业务用户**，只需要使用开发者提供的打包好程序即可，无需安装 Python、uv 等开发环境。

#### 9.1 Windows：使用打包好的 `.exe`

1. **解压发布包**
   - 通常文件名类似：`dify_chat_tester_windows_vX.Y.Z_YYYYMMDD_HHMMSS.zip`
   - 右键 → "全部解压" 到任意文件夹（例如 `D:\dify_chat_tester`）
2. **准备配置文件**
   - 在解压后的目录中找到 `.env.config.example`
   - 复制一份并重命名为 `.env.config`
   - 用记事本或 VSCode 打开 `.env.config`，按照注释填写各家 API 的地址和密钥
   - **提示**：如果需要加载外部插件，请在文件中配置 `EXTERNAL_PLUGINS_PATH`
3. **双击运行程序**
   - 在目录中找到 `dify_chat_tester.exe`
   - 双击运行即可打开终端界面
   - 根据提示选择供应商、模式等开始使用
4. **常见问题**
   - 如果 Windows 提示“来自 Internet 的应用，已被保护”：

#### 最佳实践

##### 1. 文档准备建议

- **文档质量**：确保文档内容准确、完整、清晰
- **文档结构**：使用合理的标题层级，便于 AI 理解
- **文档长度**：单个文档建议 500-3000 字，太短内容不足，太长可能超出模型上下文限制
- **文档格式**：使用标准 Markdown 格式，避免特殊字符

##### 2. 模型选择建议

| 模型          | 特点                 | 适用场景             |
| ------------- | -------------------- | -------------------- |
| qwen3-max     | 理解能力强，中文优化 | 中文知识库推荐首选   |
| deepseek-v3.2 | 推理能力强，性价比高 | 复杂文档、技术文档   |
| gpt-4o        | 质量最高，但成本高   | 重要项目、高质量要求 |
| glm-4.6       | 平衡性能和成本       | 一般场景             |

##### 3. 问题质量优化

生成的问题会自动满足以下要求：

1. 模仿真实用户提问语气，口语化
2. 覆盖文档中的各个知识点
3. 难度有所变化（简单查询、复杂推理等）
4. 每个问题都能从文档中找到答案依据
5. 每个文档至少生成 15-20 个问题

##### 4. 增量保存机制

- 程序会在每处理完一个文档后立即保存
- 即使程序中途中断，已处理的文档数据也不会丢失
- 下次可以手动删除已处理的文档，只处理新文档

#### 常见问题

##### Q1: 为什么某些文档没有生成问题？

**可能原因**：

- 文档内容太短（少于 100 字）
- 文档格式有问题（编码错误、特殊字符）
- API 调用失败

**解决方案**：

- 检查文档内容是否完整
- 确保文档使用 UTF-8 编码
- 查看控制台错误提示

##### Q2: 生成的问题质量不理想怎么办？

**优化建议**：

1. **换用更好的模型**：尝试 qwen3-max 或 gpt-4o
2. **改进文档质量**：补充更多细节，使用更清晰的表达
3. **手动筛选**：保留高质量问题，删除不合适的
4. **重新生成**：删除旧数据，调整文档后重新生成

##### Q3: 可以生成英文问题吗？

**答案**：可以，只需：

1. 使用英文的知识库文档
2. AI 会自动识别文档语言并生成对应语言的问题

##### Q4: 生成的问题太多或太少怎么办？

**调整方法**：

- 目前每个文档固定生成 15-20 个问题
- 如需调整，可以修改配置或联系开发者

## 10. ❓ 常见问题

### Q1: 提示"python 不是内部或外部命令"怎么办？

**A:** 这是正常现象，因为大多数用户安装 Python 时没有勾选 "Add Python to PATH"。

**推荐解决方案（使用 py 启动器）：**

1. **使用 py 命令代替 python**

   ```bash
   # 检查版本
   py --version

   # 运行程序
   py main.py
   ```

2. **查看所有 Python 版本**
   ```bash
   py --list
   ```

**备选解决方案：**

1. **添加环境变量**（见第 3 节详细说明）
2. **使用完整路径**（不推荐，太麻烦）

### Q2: 提示"uv 不是内部或外部命令"怎么办？

**A:** 这说明 uv 没有正确安装或没有激活虚拟环境。

**解决方案：**

1. **确保在虚拟环境中**

   ```bash
   # 激活环境
   .venv\Scripts\activate.bat

   # 检查 uv
   uv --version
   ```

2. **在虚拟环境中安装 uv**

   ```bash
   py -m pip install uv
   ```

3. **使用 py 调用 uv**
   ```bash
   py -m uv --version
   ```

### Q3: 程序启动失败怎么办？

**A:**

1. **确认 Python 版本**：

   ```bash
   python --version
   # 或
   "C:\Users\...\python.exe" --version
   ```

2. **确认 uv 已安装**：

   ```bash
   uv --version
   # 或
   python -m uv --version
   ```

3. **确认在项目目录**：

   ```bash
   dir
   # 应该看到 main.py 文件
   ```

4. **重新安装依赖**：
   ```bash
   uv sync --reinstall
   # 或
   python -m uv sync --reinstall
   ```

### Q4: API 密钥输入后看不到？

**A:**
这是正常的安全设计，密钥输入时会自动隐藏。确认时会显示前几位和后几位，中间用 `*` 替代。

### Q5: Dify 提示"模型选择"怎么办？

**A:**
Dify 不需要选择模型，程序会自动使用应用配置的模型。如果出现选择提示，可能是配置问题。

### Q6: 批量处理速度太慢？

**A:**

1. 检查网络连接
2. 调整请求间隔：在 `.env.config` 中修改 `BATCH_REQUEST_INTERVAL`
3. 减少同时处理的问题数量

### Q7: 如何查看对话历史？

**A:**
会话模式的所有对话都会自动保存在 `chat_log.xlsx` 文件中，包含完整的时间戳和对话内容。

### Q8: 程序崩溃或无响应？

**A:**

1. 按 `Ctrl+C` 强制退出
2. 检查错误日志
3. 重启程序
4. 如果问题持续，删除 `.env.config` 重新配置

### Q9: 如何更新程序？

**A:**

```bash
git pull origin main
uv sync
```

### Q10: Excel 文件打不开或损坏？

**A:**

1. 确保使用 Microsoft Excel 或兼容软件
2. 不要在程序运行时手动编辑 Excel 文件
3. 备份重要数据

### Q11: 角色如何自定义？

**A:**
在角色选择时：

- 输入预设角色编号
- 输入 `len(预设角色)+1` 选择用户角色
- 输入 `len(预设角色)+2` 自定义角色
- 或直接输入自定义角色名称

### Q12: VSCode 提示"未检测到 Python 解释器"怎么办？

**A:**

1. **手动选择 Python 解释器**

   - 按 `Ctrl+Shift+P` 打开命令面板
   - 输入 "Python: Select Interpreter"
   - 点击"选择解释器"
   - 如果没有自动检测到，点击"输入解释器路径"
   - 输入：`C:\Users\您的用户名\AppData\Local\Programs\Python\Python311\python.exe`

2. **创建 .vscode 配置**

   - 在项目根目录创建 `.vscode` 文件夹
   - 在其中创建 `settings.json` 文件：

   ```json
   {
     "python.defaultInterpreterPath": "C:\\Users\\您的用户名\\AppData\\Local\\Programs\\Python\\Python311\\python.exe"
   }
   ```

3. **重启 VSCode**
   - 关闭并重新打开 VSCode
   - VSCode 会自动检测配置

### Q13: 如何切换 AI 供应商？

**A:**

1. 退出当前模式（输入 `/quit`）
2. 程序会返回供应商选择菜单
3. 选择新的供应商并配置
